サブエージェントの作り方

Claude Code でチームを組織するための完全ガイド — 知り合いに「どうやって組織作ったの？」と聞かれたとき、これ1本で説明できる

このドキュメントは何？

Claude Code でサブエージェント（=専門家チーム）を組織するための完全ガイド。新メンバー追加するときも、知り合いに「どうやって組織作ったの？」と聞かれたときも、これ1本で説明できる。

まず最初に：2つの仕組みがある

涼晴のチームは 2階層 で動いている。混ざりやすいので最初に整理しておく。

仕組み① Claude Code 標準サブエージェント

項目内容
場所~/.claude/agents/
1人あたりMDファイル1個
用途プログラミング系の専門作業（コードレビュー、計画立案、TDD等）
起動Claudeが自動で呼ぶ。Agent ツールで明示起動も可能
例planner, code-reviewer, tdd-guide, security-reviewer

仕組み② RYOSEIWORLD AI組織メンバー（独自）

仕組み① をゼロから揃える方法

結論：素のClaude Codeには1個も入ってない

インストール直後は ~/.claude/agents/ フォルダ自体が空っぽ。仕組みは動くが、エージェントは自分で用意する必要がある。

一気に揃える：ECC（Everything Claude Code）を入れる

ECC = Anthropic ハッカソン優勝者 Affaan Mustafa が作った Claude Code 設定セット。38個のエージェント＋156個のスキル＋Hook が一発で入る。MITライセンスで無料。

リポジトリ: https://github.com/affaan-m/everything-claude-code

インストール手順（推奨：プラグイン経由）

Claude Code を起動した状態で、以下を順番に打つ：

/plugin marketplace add https://github.com/affaan-m/everything-claude-code
/plugin install everything-claude-code@everything-claude-code

これだけ。2分で終わる。

注意点

2つの方法を混ぜるな プラグイン経由でやったあとに install.sh --profile full をやると重複する。重複したら /plugin uninstall で一旦外してやり直す。

涼晴がインストールしたのは v1.9.0（2026-04-04）。最新は v2.0.0-rc.1。

入るもの（v2.0.0-rc.1時点）

  • エージェント38個（planner, code-reviewer, tdd-guide, security-reviewer 等）
  • スキル156個（/note-post, /ig-analyze, /yt-analyze 等もここから）
  • Hook（自動フォーマット、セッション保存等）
  • ルール集（言語別ベストプラクティス）

仕組み① を自分で1個書く方法

ECC入れずに、欲しいエージェントだけ自分で書く方法もある。MDファイル1個、20行で済む。

雛形

~/.claude/agents/<エージェント名>.md に以下を保存：

---
name: 俺だけのレビュアー
description: 俺の口調でコードをレビューする専門家。コード書いたあと自動で呼ばれる。
tools: ["Read", "Grep", "Glob", "Bash"]
model: sonnet
---

あなたはコードレビュアーです。

## 役割
- 俺（涼晴）が書いた・修正したコードをレビューする
- 専門用語を避けて、わかりやすい言葉で指摘する
- 致命的な問題（セキュリティ、データ消失）から先に報告する

## レビュー手順
1. `git diff` で変更内容を確認
2. セキュリティ問題を最優先でチェック
3. コードの読みやすさをチェック
4. 重要度別に報告（CRITICAL > HIGH > MEDIUM > LOW）

## 報告フォーマット
- ファイル名と行番号を必ず書く
- 何が問題か / どう直すか をセットで書く
- 1〜2文で簡潔に

frontmatter（先頭の --- で囲んだ部分）の意味

項目説明例
nameエージェントの名前。Claudeが呼ぶときに使う俺だけのレビュアー
descriptionいつ呼ぶか。Claudeはこれを読んで自動判断するコード書いたあと自動で呼ばれる
toolsこのエージェントが使えるツール。配列で書く["Read", "Grep", "Glob", "Bash"]
model使うモデル。sonnet / opus / haiku から選ぶsonnet

モデルの選び方

モデル速度賢さ値段こういうときに使う
haiku速い普通安い軽い作業（フォーマット、簡単な確認）
sonnet普通賢い普通大半の作業はこれでOK
opus遅い一番賢い高い重要な判断、複雑な計画立案
保存して即使える ファイル保存した瞬間から使える。Claude Code 再起動も不要。Claudeが description を読んで「このタスクはこのエージェントだな」と自動判断して呼ぶ。

仕組み② RYOSEIWORLD メンバーを追加する方法

新しい仲間（例：「動画編集AI」）を追加したいときの手順。

  1. フォルダを作る 
    ~/Desktop/RYOSEIWORLD_AI_Team/AI_Organization/メンバー/動画編集AI/
  2. profile.md を書く

    人格・役割・担当タスクを書く。番頭の例を真似ると早い。

    # 動画編集AI

## 役割
TikTok / YouTube Shorts の動画を編集する人。テンポ命。

## 担当タスク
- 撮影素材から短尺動画を作る
- BGM・効果音の選定
- テロップの配置
- サムネイル作成

## 参照すべきファイル
- ~/Desktop/動画素材/
- 02_RYOSEIWORLD合同会社/プロジェクト/_一覧.md

## 作業ルール
1. 1本3秒以内にフックを作る
2. テロップは最大2行まで
3. 完成したら審判AIにレビュー依頼
  3. memory.md を書く（最初は空でOK） 
    # 動画編集AI — 個人メモリ
最終更新: 2026-04-30

## 学んだこと
（ここに作業しながら学んだことを追記していく）

## 過去の成功パターン
（ここにバズった動画の特徴を記録していく）
  4. ワークフロー.md に登録

    AI_Organization/ワークフロー.md を開いて、振り分けルールに追加：

    | タスク | 担当 |
|-------|-----|
| 動画編集 | 動画編集AI |
  5. CLAUDE.md の地図にも追加

    ~/.claude/CLAUDE.md の「メンバーの記憶」テーブルに1行足す：

    | `動画編集AI/` | 動画編集・TikTok・YouTube Shorts |
これで完成 /ryoseiworld 動画編集して と言うと、自動で動画編集AIが呼ばれる。

チーム組成のコツ（涼晴の経験から）

やった方がいいこと

① 役割は1人1つに絞る 営業AIに「マーケもやって」と言うと曖昧になる。1人1機能。
② memory.md を必ず作る 学習が蓄積されるから、使えば使うほど賢くなる。
③ 審判AIを必ず通す 外部向けの成果物は、必ず審判AIにレビューさせる。これを飛ばすと事故る（2026/3/17に失敗してる）。
④ 新メンバーは月1ペースで増やす 一気に増やすと管理できない。必要になったら1人ずつ。

やっちゃダメなこと

① 同じ役割のメンバーを複数作る 「営業AI1号」「営業AI2号」みたいに分けると、どっちを呼ぶか毎回迷う。
② profile.md を放置する 役割が古くなったら更新する。古いまま使うとズレた成果物が出る。
③ ECCインストールを2回やる 重複してエージェントが二重登録されることがある。

トラブルシューティング

困ったこと対処法
エージェントが呼ばれない description が曖昧だと Claude が自動判断できない。「いつ使うか」を具体的に書く。
ECCのインストールが失敗した /plugin uninstall everything-claude-code で一旦外して、再度 /plugin install を試す。それでもダメなら GitHub Issues を確認。
メンバーを削除したい フォルダごと削除でOK。ただし ワークフロー.md と CLAUDE.md の記述も合わせて消す。
別のPCに同じ環境を作りたい ECC は再インストール。RYOSEIWORLDメンバーは AI_Organization/ フォルダごとコピー。
